關於 GrokMediaDownloader 的常見問題解答

基本功能

Generate List 按鈕是做什麼的？
Generate List 是 GrokMediaDownloader 的核心批次下載功能，這款 Chrome 擴充功能已獲得超過 8,000 位用戶信賴，專門用於管理 Grok AI 媒體。當您在 Grok 收藏頁面點擊 Generate List 時，擴充功能會透過 DOM 分析對整個頁面進行智慧掃描，偵測所有圖片和影片，包括從原始圖片生成影片的子貼文。系統會自動透過檢查本地下載歷史來過濾已下載的檔案，該歷史使用防止記憶體溢出的快取機制，可處理超過 5,000 個項目而不會當機。生成的清單只顯示全新、未下載的媒體，供您批次下載。您還可以啟用 HD 優先以偏好高畫質版本、將中繼資料儲存為 JSON 檔案，或使用提示詞作為檔名。所有處理都在您的瀏覽器中 100% 客戶端完成，無需上傳到任何伺服器。
什麼是 "Only after" 日期過濾？
Only After 日期過濾器是 GrokMediaDownloader 的 PRO 專屬功能，專為在 Grok AI 上累積了大量媒體庫的進階用戶設計。當您的收藏夾包含數千張圖片和影片時，每次全部下載既不實際又浪費頻寬。這個過濾器讓您設定一個截止日期，使 Generate List 掃描只包含在該日期之後建立的媒體。例如，您可以設定為今天只下載當前工作階段中生成的內容，或設定為本週進行每週定期批次下載。這對於每天使用 Grok 並希望維持有序下載工作流程的創作者特別有用。日期過濾器與現有的重複偵測系統配合運作，讓您精確控制哪些新作品需要下載。此功能僅適用於 $4.99 的 PRO 終身授權。
Stream Capture 與 Generate List 有什麼不同？
Generate List 和 Stream Capture 在 GrokMediaDownloader 中服務於不同的工作流程。Generate List 是一個批次掃描工具，會讀取您的 Grok 收藏頁面，找出所有圖片和影片，過濾重複項目，並建立下載佇列。您可以在想要下載累積媒體時手動執行。相比之下，Stream Capture 是一個即時監控系統，使用 MutationObserver 監聽 DOM 中新出現的圖片。當您在 grok.com 上瀏覽和創作時，Stream Capture 會在新圖片出現在畫面上的瞬間自動偵測並加入佇列，每次工作階段最多可捕獲 1,000 張圖片。簡單來說，Generate List 是從您已儲存的收藏中進行回溯式批次下載，而 Stream Capture 則是一張即時捕獲網，能在媒體產生的當下就攔截。這兩項功能都完全在客戶端運作，不會將任何資料傳送到外部伺服器。
HD Upgrade 是如何運作的？
HD Upgrade 是 GrokMediaDownloader 中的智慧影片畫質提升功能。Grok AI 有時會儲存生成影片的低解析度預覽版本以節省頻寬。HD Upgrade 會掃描您的下載清單，自動識別那些有標準畫質預覽但伺服器上有 HD 版本可用的影片，然後向 Grok 發送請求以取得 1080p 高畫質版本。系統包含智慧重複偵測功能，如果某個影片的 HD 版本已經存在，就不會發送多餘的請求。它也能識別原生 720p SuperGrok 影片並適當跳過。整個流程在啟動後自動執行，當所有影片都已被檢查並在可能的情況下升級後便會結束。HD Upgrade 與批次下載系統整合，您可以在確認升級前先預覽掃描結果。所有影片處理都在您的 Chrome 瀏覽器中 100% 客戶端完成。
我換了電腦，下載紀錄還在嗎？
預設情況下，GrokMediaDownloader 會將您的下載歷史儲存在當前電腦的 Chrome 本地儲存空間中。這意味著如果您更換到新電腦、格式化硬碟或重新安裝 Chrome，您的歷史紀錄不會自動轉移。然而，PRO 用戶可以使用 Export/Import Data 功能，將整個下載歷史匯出為檔案並在任何其他裝置上還原。這在升級電腦或維護多台工作站時進行跨裝置轉移非常實用。匯出功能會建立一個包含所有下載紀錄的可攜式資料檔案，而匯入流程則能無縫還原所有資料。由於 GrokMediaDownloader 是採用 Manifest V3 架構的 Chrome 擴充功能，100% 客戶端處理，不會將任何個人資料傳送到外部伺服器。您的歷史紀錄始終保持私密且在您的控制之下，僅儲存在本地 Chrome 儲存空間中。

Story Mode Helper

什麼是 Story Mode Helper？
Story Mode Helper 是 GrokMediaDownloader v2.0 推出的重要功能，設計為 Chrome 側邊面板中的完整影片工作室。它提供支援拖放排序的故事板管理器，讓您將多個影片片段排列成連貫的敘事序列。您可以從任何影片片段中擷取特定畫格，並使用「發送畫格到 Grok」功能將畫格連同自訂提示詞上傳到 Grok Imagine，生成延續您故事的續集影片。影片合併功能使用 FFmpeg.js，這是 FFmpeg 的 WebAssembly 實作，完全在您的瀏覽器中運行。這意味著所有影片處理，包括標準化、解析度匹配和最終合併，都在本地完成，不會上傳任何內容到伺服器。影片資料儲存在 IndexedDB 中以有效處理大型二進位檔案。Story Mode Helper 將 Grok AI 從簡單的圖片生成器轉變為完整的創意故事講述平台，一切都在您的瀏覽器中完成。
如何使用 Send Frame to Grok 功能？
「發送畫格到 Grok」是 Story Mode Helper 中的工作流程功能，連接您現有的影片片段和新的 Grok AI 生成。使用方法是先在 Chrome 側邊面板中開啟 Story Mode Helper，從故事板中選擇一個影片片段來檢視其畫格。您可以透過互動式膠捲瀏覽影片並選擇任意畫格，不僅限於最後一幀。選定要從哪個畫格繼續後，輸入描述影片下一步應該發生什麼的文字提示詞。點擊發送按鈕，擴充功能會自動將選定的畫格圖片連同您的提示詞上傳到 Grok Imagine。Grok 接著會生成一個從該畫格視覺上延續的新影片。生成的影片可以加回您的故事板進行進一步編輯或合併。這創造了一個無縫的迭代工作流程，完全在您的瀏覽器中建構更長的 AI 生成影片敘事。
影片合併是在本地還是雲端處理？
GrokMediaDownloader 中所有的影片合併都完全在您的瀏覽器本地執行。擴充功能使用 FFmpeg.js，這是業界標準 FFmpeg 影片處理庫的 WebAssembly（WASM）編譯版本。這意味著 FFmpeg 的完整功能直接在您的 Chrome 瀏覽器中作為客戶端程式碼運行，絕對不會將任何影片資料上傳到任何外部伺服器或雲端服務。當您在 Story Mode Helper 中合併影片時，FFmpeg.js 引擎會在您瀏覽器的記憶體中處理影片標準化、解析度調整、幀率匹配和最終串接。影片二進位資料在處理期間暫時儲存在 IndexedDB 中。這種方式確保了完全的隱私保護，因為您的創意內容永遠不會離開您的裝置。由 Kario LLC 發行的 GrokMediaDownloader 從一開始就採用零伺服器架構設計。這款在 Chrome Web Store 上獲得 8,000+ 用戶和 4.2/5 評分的擴充功能，所有處理都在客戶端完成。
Storyboard 中的 Master 是什麼意思？
在 GrokMediaDownloader 的 Story Mode Helper 中，Master 標記會指派給故事板序列中的第一個影片片段。Master 影片作為整個合併輸出的參考標準。當您啟動合併流程時，故事板中所有其他影片片段會自動調整解析度和幀率以匹配 Master 的規格。這確保了最終合併影片中一致的視覺品質，防止不同尺寸或速度的片段之間出現突兀的轉場。例如，如果您的 Master 是 1080p、24fps 的影片，所有後續片段在合併前都會使用 FFmpeg.js 標準化到這些確切的規格。您可以透過拖放重新排列故事板來更改哪個片段是 Master。建議選擇最高品質的片段作為 Master 以獲得最佳輸出結果。整個標準化和合併過程都透過 WebAssembly 在本地執行，不涉及任何雲端處理。
影片儲存空間有限制嗎？
GrokMediaDownloader 使用 IndexedDB（瀏覽器原生資料庫）來儲存 Story Mode Helper 的影片資料。儲存容量取決於您瀏覽器的設定和可用磁碟空間。大多數現代 Chrome 安裝允許 IndexedDB 使用從數百 MB 到數 GB 的儲存空間，超過後才會提示要求額外權限。側邊面板介面會即時顯示您目前的儲存使用量，讓您可以監控故事板影片佔用了多少空間。如果您接近儲存限制，可以匯出已完成的專案並清除較舊的影片資料以釋放空間。之所以特別選擇 IndexedDB 而非其他儲存方式，是因為它能有效處理影片檔案等大型二進位資料，而不會降低瀏覽器效能。由於 GrokMediaDownloader 中所有處理都透過其 Manifest V3 架構在客戶端完成，您的影片資料永遠不會離開本地瀏覽器儲存空間。這種設計同時確保了您創意工作的效能和隱私。

v2.2 功能

什麼是 Video Gen Queue？
Video Gen Queue 是 GrokMediaDownloader v2.2 推出的自動化影片生成排程系統，可透過 Chrome 側邊面板存取。它讓您將多個影片生成任務加入佇列，無需手動操作即可依序處理。您可以使用貼文頁面上的浮動按鈕逐一添加項目，或從最近的貼文批次添加。系統支援四種生成模式：Custom、Normal、Fun 和 Spicy，讓您控制每個生成影片的創意風格。Video Gen Queue 包含智慧錯誤處理，當 Grok 的審核系統拒絕生成時會自動重試，並能偵測速率限制（HTTP 429）回應並自動暫停以避免被節流。佇列使用 MAIN world TipTap 整合來直接與 Grok 的編輯器介面互動。免費版提供 50 次 Video Gen Queue 使用次數，而 $4.99 的 PRO 終身授權則為認真的創作者解鎖無限次使用。
什麼是 Project-Based Download？
Project-Based Download 是 GrokMediaDownloader v2.2 推出的組織功能，會自動將您的下載分組到結構化的專案資料夾中。下載前，它會掃描您的媒體並顯示完整的預下載統計資料，包括圖片數量、影片數量以及 HD 與 SD 的比例分析。當您開始下載時，系統會自動建立以 {date}_{prompt} 模式命名的專案資料夾，其中日期是媒體建立的時間，提示詞是您用來生成它的文字。每個專案資料夾還包含一個 metadata.json 檔案，記錄下載媒體的詳細資訊。這讓您的 Grok AI 創作按專案整齊組織，而非全部堆在單一資料夾中。Project-Based Download 是 PRO 專屬功能，隨 $4.99 終身授權提供。它與 GrokMediaDownloader 的其他批次下載功能並行運作，並與現有的下載歷史系統整合以防止跨專案的重複下載。

v2.3 新功能

什麼是標籤式下載？
標籤式下載是 GrokMediaDownloader v2.3 推出的功能，讓您按照 Grok 的標籤（資料夾）系統來下載媒體。不需要一次下載所有收藏，您可以在篩選區選擇「依標籤下載」，然後從下拉選單中選取特定標籤來定向下載。擴充功能接著會掃描並下載該選定標籤內的所有媒體。每個標籤維護自己獨立的下載歷史，因此您可以對不同標籤重新執行下載，無需擔心跨標籤的重複問題。下載的檔案會自動整理在電腦的 grok_downloads/tags/{標籤名稱}/ 路徑下，保持每個標籤的媒體清楚分離。這對於使用 Grok 標籤系統來組織不同專案、主題或角色合集的創作者特別有用。標籤式下載是 PRO 專屬功能，包含在 $4.99 終身授權中，專為需要精確控制下載哪些媒體類別的用戶設計。
什麼是自訂檔名模板？
自訂檔名模板是 GrokMediaDownloader v2.3 推出的功能，提供三種靈活的檔案命名方式。第一個選項是 Grok 原生格式，保留 Grok 指定的原始檔名格式。第二個選項是 Prompt 命名，使用您的生成提示詞文字作為檔名以便識別。第三個選項是自訂模板，讓您使用 {date}、{datetime}、{prompt}、{quality} 等代碼建立自由格式的模板，這些代碼在下載時會動態替換。例如，{date}_{prompt}_{quality} 模板會產生像 20260409_日落風景_HD.jpg 這樣的檔名。此功能對所有用戶開放，包括免費版和 PRO 版。如果您之前使用了較早的「Prompt 作為檔名」設定，您的偏好會自動遷移到新的 Prompt 命名選項。自訂檔名模板適用於所有下載模式，包括批次下載、Stream Capture 和標籤式下載。
Story Mode 的 Frame Lock 是什麼？
Frame Lock 是 GrokMediaDownloader v2.3 推出的 Story Mode 增強功能，讓您精確控制影片延續點。當使用「發送畫格到 Grok」生成續集影片時，您現在可以透過互動式膠捲從影片片段中選擇任意畫格，而不僅是預設的最後一幀。這讓您可以選擇希望 Grok 從哪個確切的視覺時刻開始延續。在您發送畫格並且 Grok 生成續集影片後，選定的畫格會在膠捲上以橘色指示器鎖定，標記轉場點的位置。當您匯出或合併最終影片時，FFmpeg.js 會自動在鎖定畫格位置裁剪來源片段。這可以防止連續片段之間出現重疊或重複的內容，確保合併的故事板輸出中有無縫的視覺轉場。Frame Lock 消除了之前需要的手動裁剪，使迭代式影片創作工作流程對故事創作者來說顯著更加順暢。
