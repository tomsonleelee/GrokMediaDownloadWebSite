核心功能

串流擷取提供無縫的即時圖片收集工作流程

即時 DOM 觀察者

使用 MutationObserver 監控 Grok 生成的新圖片。即時偵測，無需手動重新整理。

自動排入佇列

每張偵測到的圖片會自動加入下載佇列。只需繼續滾動 — 串流擷取會處理其餘的事。

一鍵批次下載

瀏覽結束後，一次下載所有擷取的圖片。不用再一張一張儲存。

智慧去重

使用雜湊比對自動過濾收藏中已存在的重複圖片。

串流擷取運作方式

四個簡單步驟，自動擷取每一張 Grok 圖片

1

啟動監控

開啟擴充功能，點擊「啟動串流擷取」開始即時監控 Grok 頁面。

2

瀏覽與滾動

正常瀏覽 Grok。頁面上出現的每張新圖片都會被自動偵測並排入佇列。

3

檢視已擷取

查看擷取佇列中所有自動捕捉的圖片。移除不需要的項目。

4

批次下載

點擊「全部下載」，一次將所有擷取的圖片儲存到本地磁碟。

技術亮點

MutationObserver API

使用瀏覽器原生的 MutationObserver 高效偵測 DOM 變更，不影響效能。

背景 Service Worker

在 Chrome 的 Service Worker 中運行，即使切換分頁也能持續監控。

IndexedDB 快取

將擷取的圖片中繼資料儲存在 IndexedDB 中，用於快速去重檢查和佇列管理。

隱私優先

所有圖片偵測和下載都在本地進行。不會向任何外部伺服器傳送資料。

常見問題

關於 Stream Capture 的所有問題解答

Stream Capture 能在所有 Grok 頁面上運作嗎？

可以。Stream Capture 監控任何 Grok 產生圖片的頁面，包含主頁面、個別貼文頁面及搜尋結果。採用 DOM 觀察技術，只要 Grok 渲染新圖片，就能自動偵測。

一次監控可以捕捉多少張圖片？

沒有數量上限。Stream Capture 可以在單次監控期間排入數百張圖片。佇列儲存於 IndexedDB，即使切換分頁或關閉擴充功能面板也不會遺失。

Stream Capture 會讓瀏覽器變慢嗎？

不會。Stream Capture 使用 Chrome 原生的 MutationObserver API，效率極高。觀察器只在 DOM 變更時才會觸發，在沒有主動產生圖片時 CPU 使用率接近零。背景 Service Worker 負責佇列管理，不會阻塞主執行緒。

可以暫停和恢復監控嗎？

可以。在擴充功能面板中點擊「停止監控」即可暫停。之前捕捉的圖片仍保留在佇列中。再次點擊「開始 Stream Capture」即可繼續，新圖片會加入同一佇列。

Stream Capture 會自動去除重複圖片嗎？

會。Stream Capture 使用雜湊比對來偵測並跳過已在收藏或目前佇列中的圖片，避免在上滑或重新瀏覽貼文時重複下載同一張圖片。

Stream Capture 支援哪些圖片格式？

Stream Capture 以 Grok 提供的原始格式下載圖片，通常為 JPEG 和 PNG。下載過程中不會對圖片進行重新編碼或壓縮，保留原始解析度和品質。

Stream Capture 是免費功能嗎？

Stream Capture 是 PRO 專屬功能。免費版支援可見圖片的手動批次下載。升級至 PRO（$4.99 終身授權）可解鎖 Stream Capture、Video Gen Queue、Story Mode 和 Project Download。
