### Critical Performance Issue: Cache Headers

**All static resources serve with `Cache-Control: public, max-age=0, must-revalidate`.**

This means every page visit requires revalidation of CSS, JS, and images. For a static site with infrequently changing assets, this causes unnecessary network requests and degrades repeat visit performance.

**Recommended fix in `vercel.json`:**

```json
{
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/css/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/js/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=2592000" }
      ]
    }
  ]
}
```

### Font Loading

Loading 3 Google Font families in a single request:
- `Inter:wght@400;600;800`
- `Noto+Sans+TC:wght@300;400;500;700`
- `Noto+Sans+JP:wght@300;400;500;700`

This creates a render-blocking request chain: HTML → fonts.googleapis.com (CSS) → fonts.gstatic.com (WOFF2 files). The `display=swap` parameter mitigates FOIT (Flash of Invisible Text) but doesn't eliminate the network cost.

**Recommendation:** Consider self-hosting fonts or using `<link rel="preconnect">` for `fonts.googleapis.com` and `fonts.gstatic.com` (check if already present).
