# i18n Rules

## Core Principle

Every text change MUST be applied to all three languages simultaneously. Body text is **hardcoded in HTML** — no JS dependency for SEO.

## Architecture

- **Body text**: Hardcoded in each HTML file in the correct language
- **Header/Footer**: Language-specific component files (`header.html`, `header-zh-TW.html`, `header-ja.html`, same for footer)
- **`js/i18n.js`**: Only contains toast translations (4 keys × 3 languages) + language toggle + path detection
- **`data-i18n` attributes**: Remain in HTML as CSS/JS selectors, but text is already the correct language

## URL Structure

| Language | Path | HTML `lang` | Note |
|----------|------|-------------|------|
| English | `/` (root) | `en` | Default & x-default |
| 繁體中文 | `/zh-TW/` | `zh-TW` | |
| 日本語 | `/ja/` | `ja` | |

## HTML Page Three-Language Sync

When editing an HTML page, the same structural change must be applied to all three versions:

| Element | EN (root) | zh-TW (`/zh-TW/`) | ja (`/ja/`) |
|---------|-----------|---------------------|-------------|
| `<html lang>` | `en` | `zh-TW` | `ja` |
| `<title>` | English title | 中文標題 | 日本語タイトル |
| `<meta description>` | English desc | 中文描述 | Japanese desc |
| `<link canonical>` | Root URL | `/zh-TW/` URL | `/ja/` URL |
| `<link hreflang>` | All three + x-default | All three + x-default | All three + x-default |
| Body text | English | 繁體中文 | 日本語 |
| Internal links | Root paths | `/zh-TW/` paths | `/ja/` paths |

## Adding New Text Content

1. Add the English text directly in the root HTML file
2. Add the Traditional Chinese text in `zh-TW/` version
3. Add the Japanese text in `ja/` version
4. Optionally add `data-i18n="key_name"` attribute for JS selector use (not required for translation)

## Component Sync

When modifying header or footer, update all 3 versions:
- `components/header.html` (EN) + `components/header-zh-TW.html` + `components/header-ja.html`
- `components/footer.html` (EN) + `components/footer-zh-TW.html` + `components/footer-ja.html`

## Common Mistakes to Avoid

- Changing HTML structure in one language version but not others → layout inconsistency
- Forgetting to update component files when changing header/footer
- Adding JS-dependent text instead of hardcoded text → Googlebot won't see it
- Using wrong link prefix (root pages use `/`, zh-TW uses `/zh-TW/`, ja uses `/ja/`)
