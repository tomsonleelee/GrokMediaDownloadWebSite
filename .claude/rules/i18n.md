# i18n Rules

## Core Principle

Every text change MUST be applied to all three languages simultaneously. Never leave a language block out of sync.

## Translation Key Naming

- Format: `snake_case`
- Pattern: `{section}_{element}_{detail}`
- Section prefixes:

| Prefix | Section |
|--------|---------|
| `nav_` | Navigation |
| `hero_` | Hero section |
| `queue_` | Video Gen Queue |
| `story_` | Story Mode |
| `studio_` | Video Studio |
| `stream_` | Stream Capture |
| `feat_` | Features |
| `pain_` | Pain points |
| `price_` | Pricing |
| `faq_` | FAQ |
| `footer_` | Footer |
| `contact_` | Contact modal |
| `legal_` | Legal pages |
| `toast_` | Toast notifications |
| `perf_` | Performance notes |
| `success_` | Success page |

## Adding a New Translation Key

1. Open `js/i18n.js`
2. Add the key to the `zh-TW` block with Traditional Chinese text
3. Add the same key to the `en` block with English text
4. Add the same key to the `ja` block with Japanese text
5. In the HTML, use `data-i18n="your_key_name"` on the target element
6. Verify the key count stays consistent across all three language blocks

## HTML Page Three-Language Sync

When editing an HTML page, the same structural change must be applied to all three versions:

| Element | zh-TW (root) | en (`/en/`) | ja (`/ja/`) |
|---------|--------------|-------------|-------------|
| `<html lang>` | `zh-TW` | `en` | `ja` |
| `<title>` | 中文標題 | English title | 日本語タイトル |
| `<meta description>` | 中文描述 | English desc | Japanese desc |
| `<link canonical>` | Root URL | `/en/` URL | `/ja/` URL |
| `<link hreflang>` | All three | All three | All three |
| Content structure | Identical | Identical | Identical |

## Common Mistakes to Avoid

- Adding a key to one language block but forgetting others → runtime shows raw key name
- Changing HTML structure in one language version but not others → layout inconsistency
- Using hardcoded text instead of `data-i18n` attributes → text won't switch on language change
