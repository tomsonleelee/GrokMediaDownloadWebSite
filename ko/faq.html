기본 기능

Generate List란 무엇인가요?
Generate List는 GrokMediaDownloader의 핵심 일괄 다운로드 기능입니다. 8,000명 이상의 사용자가 신뢰하는 이 Chrome 확장 프로그램은 Grok AI 미디어 관리에 특화되어 있습니다. Grok 즐겨찾기 페이지에서 Generate List를 클릭하면 확장 프로그램이 DOM 분석을 통해 전체 페이지를 지능적으로 스캔하여 모든 이미지와 동영상을 감지합니다. 소스 이미지에서 생성된 동영상을 포함한 하위 게시물도 대상입니다. 시스템은 로컬 다운로드 이력을 확인하여 이미 다운로드된 파일을 자동으로 필터링합니다. 이 이력은 OOM 안전 캐싱을 사용하여 5,000개 이상의 항목을 충돌 없이 처리할 수 있습니다. 생성된 목록에는 새로운 미다운로드 미디어만 표시됩니다. HD 우선을 활성화하여 고화질 버전을 우선하거나, 메타데이터를 JSON 파일로 저장하거나, 프롬프트 텍스트를 파일명으로 사용할 수 있습니다. 모든 처리는 브라우저에서 100% 클라이언트 사이드로 수행됩니다.
Only after 날짜 필터란?
Only After 날짜 필터는 GrokMediaDownloader의 PRO 전용 기능으로, Grok AI에서 대량의 미디어 라이브러리를 축적한 파워 유저를 위해 설계되었습니다. 즐겨찾기에 수천 장의 이미지와 동영상이 있을 때 매번 전부 다운로드하는 것은 비실용적이고 대역폭 낭비입니다. 이 필터를 사용하면 특정 기준 날짜를 설정하여 Generate List 스캔이 해당 날짜 이후에 생성된 미디어만 포함하도록 할 수 있습니다. 예를 들어, 오늘로 설정하면 현재 세션에서 생성한 것만 다운로드하거나, 이번 주로 설정하면 주간 정기 일괄 다운로드 루틴을 만들 수 있습니다. 매일 Grok을 사용하며 체계적인 다운로드 워크플로를 유지하려는 크리에이터에게 특히 유용합니다. 날짜 필터는 기존 중복 감지 시스템과 함께 작동하여 어떤 새 작품을 다운로드할지 정확하게 제어할 수 있습니다. $4.99 PRO 평생 라이선스로 이용 가능합니다.
Stream Capture와 Generate List의 차이점은?
Generate List와 Stream Capture는 GrokMediaDownloader에서 서로 다른 워크플로를 제공합니다. Generate List는 Grok 즐겨찾기 페이지를 읽고 모든 이미지와 동영상을 찾아 중복을 필터링하고 다운로드 큐를 생성하는 일괄 스캔 도구입니다. 축적된 미디어를 다운로드하고 싶을 때 수동으로 실행합니다. 반면 Stream Capture는 MutationObserver를 사용하여 DOM에서 새로 나타나는 이미지를 감시하는 실시간 모니터링 시스템입니다. Stream Capture를 활성화하면 grok.com에서 브라우징하고 제작하는 동안 새 이미지가 화면에 나타나는 순간 자동으로 감지하고 큐에 추가하며, 세션당 최대 1,000장의 이미지를 캡처할 수 있습니다. Generate List는 저장된 컬렉션에서의 소급 일괄 다운로드이고, Stream Capture는 발생하는 모든 것을 실시간으로 캡처하는 라이브 네트라고 생각하면 됩니다. 두 기능 모두 클라이언트 사이드에서 완전히 작동하며 외부 서버로 데이터를 전송하지 않습니다.
HD Upgrade는 어떻게 작동하나요?
HD Upgrade는 GrokMediaDownloader의 스마트 동영상 화질 향상 기능입니다. Grok AI는 대역폭 절약을 위해 생성된 동영상의 저해상도 미리보기 버전을 저장하는 경우가 있습니다. HD Upgrade는 다운로드 목록을 스캔하여 표준 화질 미리보기는 있지만 서버에 HD 버전이 존재하는 동영상을 자동으로 식별하고, Grok에 1080p 고화질 버전 요청을 보냅니다. 지능형 중복 감지 기능이 포함되어 있어 특정 동영상의 HD 버전이 이미 존재하면 불필요한 요청을 보내지 않습니다. 네이티브 720p SuperGrok 동영상도 적절하게 인식하고 건너뜁니다. 전체 프로세스는 시작 후 자동으로 실행되며, 목록의 모든 동영상이 확인되고 가능한 경우 업그레이드되면 종료됩니다. HD Upgrade는 일괄 다운로드 시스템과 연동되어 업그레이드 전에 미리보기 확인 스캔이 가능합니다. 모든 동영상 처리는 Chrome 브라우저 내에서 100% 클라이언트 사이드로 수행됩니다.
PC를 교체하면 이력이 사라지나요?
기본적으로 GrokMediaDownloader는 현재 컴퓨터의 Chrome 로컬 스토리지에 다운로드 이력을 저장합니다. 따라서 새 컴퓨터로 교체하거나, 드라이브를 포맷하거나, Chrome을 재설치하면 이력이 자동으로 이전되지 않습니다. 그러나 PRO 사용자는 Export/Import Data 기능에 접근할 수 있어 전체 다운로드 이력을 파일로 백업하고 다른 기기에서 복원할 수 있습니다. 컴퓨터 업그레이드나 여러 워크스테이션 유지 시 크로스 디바이스 전송에 유용합니다. 내보내기는 모든 다운로드 기록을 포함하는 이동 가능한 데이터 파일을 생성하고, 가져오기 프로세스는 모든 것을 매끄럽게 복원합니다. GrokMediaDownloader는 Manifest V3 Chrome 확장 프로그램으로 100% 클라이언트 사이드 처리로 구축되어 있어 개인 데이터가 외부 서버로 전송되는 일이 절대 없습니다. 이력은 항상 비공개로 유지되며 로컬 Chrome 스토리지에만 저장됩니다.

Story Mode Helper

Story Mode Helper란?
Story Mode Helper는 GrokMediaDownloader v2.0에서 도입된 주요 기능으로, Chrome 사이드 패널 내의 완전한 동영상 스튜디오로 설계되었습니다. 드래그 앤 드롭 재정렬을 지원하는 스토리보드 매니저를 제공하여 여러 동영상 클립을 일관된 내러티브 시퀀스로 배치할 수 있습니다. 임의의 동영상 세그먼트에서 특정 프레임을 추출하고, Send Frame to Grok 기능으로 프레임과 커스텀 프롬프트를 Grok Imagine에 업로드하여 스토리를 확장하는 후속 동영상을 생성할 수 있습니다. 동영상 병합 기능은 FFmpeg의 WebAssembly 구현인 FFmpeg.js를 사용하며 완전히 브라우저 내에서 실행됩니다. 이는 정규화, 해상도 매칭, 최종 병합을 포함한 모든 동영상 처리가 로컬에서 이루어지며 서버 업로드가 없음을 의미합니다. 동영상 데이터는 대용량 바이너리의 효율적 처리를 위해 IndexedDB에 저장됩니다. Story Mode Helper는 Grok AI를 단순한 이미지 생성기에서 브라우저 내 완전한 크리에이티브 스토리텔링 플랫폼으로 변화시킵니다.
Send Frame to Grok 사용법은?
Send Frame to Grok은 Story Mode Helper의 워크플로 기능으로, 기존 동영상 클립과 새로운 Grok AI 생성을 연결합니다. 사용하려면 먼저 Chrome 사이드 패널에서 Story Mode Helper를 엽니다. 스토리보드에서 동영상 세그먼트를 선택하여 프레임을 봅니다. 인터랙티브 필름스트립으로 동영상을 탐색하며 마지막 프레임뿐만 아니라 원하는 프레임을 선택할 수 있습니다. 이어서 생성하고 싶은 프레임을 선택한 후, 동영상에서 다음에 일어날 내용을 설명하는 텍스트 프롬프트를 입력합니다. 전송 버튼을 클릭하면 확장 프로그램이 선택한 프레임 이미지와 프롬프트를 Grok Imagine에 자동으로 업로드합니다. Grok은 해당 프레임에서 시각적으로 이어지는 새 동영상을 생성합니다. 생성된 동영상은 스토리보드에 다시 추가하여 추가 편집이나 병합이 가능합니다. 이를 통해 브라우저 내에서 완전히 AI 생성 장편 동영상 내러티브를 구축하는 매끄러운 반복 워크플로가 실현됩니다.
동영상 병합은 로컬인가요, 클라우드인가요?
GrokMediaDownloader의 모든 동영상 병합은 완전히 브라우저 내 로컬에서 수행됩니다. 확장 프로그램은 업계 표준 FFmpeg 동영상 처리 라이브러리의 WebAssembly(WASM) 컴파일 버전인 FFmpeg.js를 사용합니다. 이는 FFmpeg의 전체 기능이 Chrome 브라우저 내에서 클라이언트 사이드 코드로 직접 실행되며, 동영상 데이터가 외부 서버나 클라우드 서비스에 업로드되는 일이 절대 없음을 의미합니다. Story Mode Helper에서 동영상을 병합할 때 FFmpeg.js 엔진은 브라우저 메모리 내에서 동영상 정규화, 해상도 조정, 프레임 레이트 매칭, 최종 연결을 모두 처리합니다. 동영상 블롭은 처리 중 IndexedDB에 임시 저장됩니다. 이 접근 방식은 크리에이티브 콘텐츠가 기기를 벗어나지 않으므로 완전한 프라이버시를 보장합니다. Kario LLC가 배포하는 GrokMediaDownloader는 제로 서버 아키텍처로 설계되었습니다. Chrome Web Store에서 8,000명 이상의 사용자와 4.2/5 평점을 받은 이 확장 프로그램은 모든 것을 클라이언트 사이드에서 처리합니다.
Storyboard의 "Master"란?
GrokMediaDownloader의 Story Mode Helper에서 Master 지정은 스토리보드 시퀀스의 첫 번째 동영상 클립에 할당됩니다. Master 동영상은 전체 병합 출력의 참조 기준이 됩니다. 병합 프로세스를 시작하면 스토리보드의 다른 모든 동영상 클립은 Master의 사양에 맞게 해상도와 프레임 레이트가 자동으로 조정됩니다. 이를 통해 최종 병합 동영상 전체에서 일관된 시각적 품질이 유지되어 서로 다른 크기나 속도의 클립 간 부자연스러운 전환을 방지합니다. 예를 들어 Master가 1080p, 24fps 동영상인 경우 모든 후속 클립은 병합 전에 FFmpeg.js를 사용하여 정확히 해당 사양으로 정규화됩니다. 스토리보드를 드래그 앤 드롭으로 재정렬하여 Master를 변경할 수 있습니다. 최고 품질의 클립을 Master로 선택하는 것이 최상의 출력 결과를 위해 권장됩니다. 이 전체 정규화 및 병합 프로세스는 WebAssembly를 통해 로컬에서 실행되며 클라우드 처리는 관여하지 않습니다.
동영상 저장 용량에 제한이 있나요?
GrokMediaDownloader는 Story Mode Helper의 동영상 데이터를 저장하기 위해 브라우저 네이티브 데이터베이스인 IndexedDB를 사용합니다. 스토리지 용량은 브라우저 설정과 사용 가능한 디스크 공간에 따라 달라집니다. 대부분의 최신 Chrome 환경에서 IndexedDB는 추가 권한을 요청하기 전에 수백 MB에서 수 GB의 스토리지를 사용할 수 있습니다. 사이드 패널 인터페이스에 현재 스토리지 사용량이 실시간으로 표시되어 스토리보드 동영상이 얼마나 많은 공간을 차지하는지 모니터링할 수 있습니다. 스토리지 한계에 가까워지면 완료된 프로젝트를 내보내고 오래된 동영상 데이터를 삭제하여 공간을 확보할 수 있습니다. IndexedDB가 다른 스토리지 방법 대신 선택된 이유는 동영상 파일과 같은 대용량 바이너리 블롭을 브라우저 성능 저하 없이 효율적으로 처리할 수 있기 때문입니다. GrokMediaDownloader의 모든 처리는 Manifest V3 아키텍처를 통해 클라이언트 사이드에서 이루어지므로 동영상 데이터가 로컬 브라우저 스토리지를 벗어나지 않습니다.

v2.2 기능

Video Gen Queue란?
Video Gen Queue는 GrokMediaDownloader v2.2에서 도입된 자동 동영상 생성 스케줄링 시스템으로, Chrome 사이드 패널에서 접근할 수 있습니다. 여러 동영상 생성 작업을 큐에 추가하고 수동 개입 없이 순차적으로 처리할 수 있습니다. 게시물 페이지의 플로팅 버튼으로 개별 추가하거나 최근 게시물에서 일괄 추가할 수 있습니다. Custom, Normal, Fun, Spicy 4가지 생성 모드를 지원하여 각 생성 동영상의 크리에이티브 스타일을 제어할 수 있습니다. Video Gen Queue에는 Grok의 모더레이션 시스템이 생성을 거부할 때 자동 재시도하는 스마트 오류 처리가 포함되어 있으며, 레이트 리밋(HTTP 429) 응답을 감지하여 스로틀링을 방지하는 자동 일시정지 기능이 있습니다. 큐는 MAIN world TipTap 통합을 사용하여 Grok의 에디터 인터페이스와 직접 상호작용합니다. 무료 버전은 50회 사용 가능하며, $4.99 PRO 평생 라이선스로 무제한 사용이 해제됩니다.
Project-Based Download란?
Project-Based Download는 GrokMediaDownloader v2.2에서 도입된 정리 기능으로, 다운로드를 구조화된 프로젝트 폴더에 자동으로 그룹화합니다. 다운로드 전에 미디어를 스캔하여 이미지 수, 동영상 수, HD 대 SD 비율을 포함한 종합적인 다운로드 전 통계를 표시합니다. 다운로드를 진행하면 {날짜}_{프롬프트} 패턴으로 프로젝트 폴더가 자동 생성됩니다. 날짜는 미디어가 생성된 날짜이고 프롬프트는 생성에 사용한 텍스트입니다. 각 프로젝트 폴더에는 다운로드된 미디어의 상세 정보를 담은 metadata.json 파일도 포함됩니다. 이를 통해 Grok AI 작품이 하나의 폴더에 무분별하게 쌓이는 대신 프로젝트별로 깔끔하게 정리됩니다. Project-Based Download는 $4.99 평생 라이선스로 이용할 수 있는 PRO 전용 기능입니다. GrokMediaDownloader의 다른 일괄 다운로드 기능과 함께 작동하며, 프로젝트 간 중복을 방지하는 기존 다운로드 이력 시스템과 통합됩니다.

v2.3 신기능

태그별 다운로드란?
태그별 다운로드는 GrokMediaDownloader v2.3에서 도입된 기능으로, Grok의 태그(폴더) 시스템에 따라 미디어를 다운로드할 수 있습니다. 모든 즐겨찾기를 한 번에 다운로드하는 대신 필터 섹션에서 '태그로 다운로드'를 선택하고 드롭다운 메뉴에서 태그를 선택하여 특정 태그를 대상으로 할 수 있습니다. 확장 프로그램은 선택한 태그 내의 모든 미디어를 스캔하고 다운로드합니다. 각 태그는 자체적인 독립 다운로드 이력을 유지하므로 태그 간 중복 걱정 없이 다른 태그의 다운로드를 재실행할 수 있습니다. 다운로드된 파일은 컴퓨터의 grok_downloads/tags/{태그명}/ 하위에 자동으로 정리되어 각 태그의 미디어가 깔끔하게 분리됩니다. Grok의 태그 시스템을 사용하여 다양한 프로젝트, 테마, 캐릭터 컬렉션을 정리하는 크리에이터에게 특히 유용합니다. 태그별 다운로드는 $4.99 평생 라이선스에 포함된 PRO 전용 기능입니다.
커스텀 파일명 템플릿이란?
커스텀 파일명 템플릿은 GrokMediaDownloader v2.3에서 도입된 기능으로, 다운로드 파일 이름 지정에 세 가지 유연한 방법을 제공합니다. 첫 번째 옵션은 Grok 기본으로, Grok이 할당한 원래 파일명 형식을 유지합니다. 두 번째는 프롬프트 기반으로, 생성 프롬프트 텍스트를 파일명으로 사용하여 쉽게 식별할 수 있습니다. 세 번째는 커스텀으로, {date}, {datetime}, {prompt}, {quality} 등의 토큰을 사용한 자유 형식 템플릿을 생성할 수 있으며 다운로드 시 동적으로 대체됩니다. 예를 들어 {date}_{prompt}_{quality} 템플릿은 20260409_sunset-landscape_HD.jpg와 같은 파일명을 생성합니다. 이 기능은 무료 버전과 PRO 버전 모든 사용자가 이용 가능합니다. 이전에 '프롬프트를 파일명으로' 설정을 사용했다면 템플릿 시스템의 새 프롬프트 기반 옵션으로 자동 마이그레이션됩니다. 커스텀 파일명 템플릿은 일괄 다운로드, Stream Capture, 태그별 다운로드 등 모든 다운로드 모드에서 작동합니다.
Story Mode의 Frame Lock이란?
Frame Lock은 GrokMediaDownloader v2.3에서 도입된 Story Mode 향상 기능으로, 동영상 연속 지점을 정밀하게 제어할 수 있습니다. Send Frame to Grok으로 후속 동영상을 생성할 때 기본 마지막 프레임뿐만 아니라 인터랙티브 필름스트립에서 동영상 클립의 임의 프레임을 선택할 수 있습니다. 이를 통해 Grok이 이어서 만들기를 원하는 정확한 시각적 순간을 선택할 수 있습니다. 프레임을 전송하고 Grok이 후속 동영상을 생성한 후 선택된 프레임은 필름스트립에 주황색 표시로 잠겨 전환 지점을 나타냅니다. 최종 동영상을 내보내거나 병합할 때 FFmpeg.js가 잠긴 프레임 위치에서 소스 클립을 자동으로 트리밍합니다. 이는 연속 클립 간 중복이나 반복 콘텐츠를 방지하여 병합된 스토리보드 출력에서 매끄러운 시각적 전환을 보장합니다. Frame Lock은 이전에 필요했던 수동 트리밍을 제거하고 반복적 동영상 제작 워크플로를 크게 개선합니다.

Generate List를 사용하여 Grok 즐겨찾기 페이지를 스캔하고 완전한 일괄 다운로드 지원으로 이미지를 다운로드할 수 있습니다. 무료 버전에는 기본 Stream Capture 기능과 FFmpeg.js WebAssembly 기반의 핵심 스토리보드 및 동영상 병합 기능을 갖춘 Story Mode Helper 접근도 포함됩니다. Video Gen Queue는 50회까지 사용 가능합니다. 커스텀 파일명 템플릿의 세 가지 명명 모드도 무료 버전에 포함됩니다. 그러나 고급 기능에는 PRO 라이선스가 필요합니다. 대상 다운로드를 위한 Only After 날짜 필터, 자동 동영상 화질 향상을 위한 HD Upgrade, 정리된 폴더 구조를 위한 Project-Based Download, 태그별 미디어 검색을 위한 태그별 다운로드, 크로스 디바이스 이력 전송을 위한 Export/Import Data, 무제한 Video Gen Queue 사용이 해당됩니다.
